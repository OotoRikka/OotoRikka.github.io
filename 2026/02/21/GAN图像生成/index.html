文件一：训练脚本（GAN_train.py）

1. 导入库

  • torch：PyTorch 核心库，提供张量操作、自动求导、GPU 支持.
  • torch.nn：神经网络模块，包含层（卷积、全连接）、损失函数、容器.
  • torchvision：处理图像的工具库，包含常用数据集、图像变换、保存图像等功能.
  • os：操作系统接口，用于创建文件夹、处理路径.
  • DataLoader：数据加载器，将数据集包装为可迭代的批量数据，支持多进程、打乱等.
  • tqdm：进度条工具，显示训练迭代进度.

2. 参数配置类 Config

  • 所有超参数集中管理，方便修改。
  • noise_dim：生成器接收的随机噪声向量的长度.
  • feats_channel：控制网络中特征图的通道数基数，后续层会乘以倍数（8,4,2,1）.

3. 生成网络 Gnet

结构解析：

  • 输入：噪声张量形状为 (batch_size, noise_dim, 1, 1)。
  • 每一层转置卷积（又称解卷积）负责将特征图尺寸扩大，通道数逐渐减少，最终生成与真实图像相同尺寸（96×96）的RGB图像.
  • 转置卷积尺寸计算公式：
  • 具体计算：
    • 第1层：输入1×1，kernel=4, stride=1, padding=0 → 输出4×4，通道512（64×8）.
    • 第2层：输入4×4，kernel=4, stride=2, padding=1 → 输出8×8，通道256.
    • 第3层：输入8×8 → 输出16×16，通道12.
    • 第4层：输入16×16 → 输出32×32，通道64.
    • 第5层：输入32×32，kernel=5, stride=3, padding=1 → 输出96×96，通道3.
  • 批归一化（BatchNorm）加速训练收敛，稳定梯度.
  • ReLU激活函数（最后一层用Tanh）引入非线性，Tanh将像素值映射到[-1,1]，与ToTensor归一化后的真实图像范围一致.

4. 判别网络 Dnet

结构解析：

  • 输入：真实或生成图像，尺寸 (batch_size, 3, 96, 96)。
  • 普通卷积逐步下采样，提取特征，最后输出一个标量概率（通过Sigmoid压缩到[0,1]）.

  • 尺寸计算公式：

  • 具体计算：

    • 第1层：输入96×96，kernel=5, stride=3, padding=1 → 输出32×32，通道64.
    • 第2层：输入32×32，kernel=4, stride=2, padding=1 → 输出16×16，通道128.
    • 第3层：输入16×16 → 输出8×8，通道256.
    • 第4层：输入8×8 → 输出4×4，通道512.
    • 第5层：输入4×4，kernel=4, stride=1, padding=0 → 输出1×1，通道1 → 经过Sigmoid得到概率.
  • 使用LeakyReLU（负斜率0.2）防止神经元死亡，缓解梯度消失.
  • 批归一化（除第一层外）稳定训练.
  • view(-1)将形状 (batch_size,1,1,1) 变为 (batch_size,)，即变成一个矩阵，每行对应一张图片的各个参数，总共有batch_size行，方便与标签计算损失.

5. 数据准备

  • ImageFolder要求数据按文件夹组织，每个子文件夹代表一个类别，自动分配标签。对于人脸数据集，可能只有一个子文件夹.
  • 变换步骤：先Resize到指定尺寸，再中心裁剪保证图像尺寸一致，最后转为Tensor（值范围[0,1]）.
  • num_workers：子进程数，0表示主进程加载。若CPU性能足够可设为>0，提高数据加载并行度.
  • pin_memory=True：将张量固定在内存中，使GPU能更快地访问.
  • prefetch_factor：每个worker预先加载的批次数.

6. 设备与网络实例化

7. 优化器与损失函数

  • Adam优化器，学习率2e-4，β1=0.5（DCGAN推荐，有助于稳定训练）.
  • 损失函数：二分类交叉熵（Binary Cross Entropy），适用于判别器输出概率与标签之间的比较.

8. 标签与噪声

  • true_labels：真实图像对应的目标概率（1），fake_labels：生成图像对应的目标概率（0）.
  • noises：训练过程中动态更新的噪声，用于生成假图像.
  • test_noises：固定的噪声，用于每个epoch结束后生成示例图像，便于观察生成效果.

9. 加载已有模型（可选）

  • 如果之前保存过权重，则加载继续训练；否则从头开始.

10. 训练循环

训练逻辑：

  • 每个iteration中，判别器和生成器各更新一次（可修改比例）.
  • 判别器训练：
    • 输入真实图像，计算输出与真实标签（1）的损失 d_real_loss..
    • 平均两者作为判别器总损失，反向传播更新判别器参数.
  • 生成器训练：
    • 更新噪声（重新采样），生成假图像.
    • 将假图像送入判别器，目标是让判别器输出接近1（即欺骗判别器），计算损失 g_loss.
    • 反向传播更新生成器参数.
  • 验证：每个epoch后用固定噪声 test_noises 生成一批图像，保存前16张，观察生成质量.
  • 保存模型：保存判别器和生成器的权重，以便恢复训练或推理.

文件二：推理脚本（inference.py）

  • 加载训练好的生成器，输入随机噪声，生成单张图像并保存.
  • g_net.eval() 很重要，因为生成器中包含BatchNorm，在推理时应使用全局统计量而非当前batch统计量.

神经网络结构

1. 生成器使用转置卷积逐步上采样

  • 传统GAN生成器通常使用全连接层将噪声映射为图像，但全连接层会丢失空间结构信息。DCGAN首次引入全卷积架构，通过转置卷积将低维噪声逐步“解码”为高分辨率图像，保留了空间相关性，生成图像质量更高.
  • 步长>1的转置卷积相当于上采样+卷积，能学习到更丰富的上采样模式，而非简单的插值.
  • 通道数递减：浅层特征图通道多、尺寸小，负责捕捉整体结构；深层特征图通道少、尺寸大，负责精细纹理。这种金字塔结构与图像的多尺度特性吻合.

2. 判别器使用步长卷积下采样

  • 用带步长的卷积代替池化层，使网络自动学习下采样方式，保留更多信息
  • 逐渐增加通道数，捕获越来越抽象的特征，最后通过单个神经元输出全局真伪概率.

3. 批归一化（BN）的作用

  • 缓解内部协变量偏移，加速训练收敛.
  • 允许使用更高的学习率，减少对初始化敏感度.
  • 在生成器中，BN有助于防止模式崩溃，使梯度传播更稳定.
  • 在判别器中，BN防止过拟合，提高泛化能力。但第一层通常不加BN，以免影响原始输入分布.

4. 激活函数选择

  • 生成器：隐藏层用ReLU，梯度不易饱和，计算高效；输出层用Tanh，将像素值约束到[-1,1]，与真实图像经ToTensor后的[0,1]范围不同，但可以通过调整真实图像预处理匹配[-1,1]（本代码未做，但Tanh仍适用，因为损失函数希望生成图像与真实图像分布相近，而真实图像经ToTensor为[0,1]，但判别器输入是真实图像也是[0,1]，所以生成器输出通过Tanh到[-1,1]后，与真实图像分布不一致？实际上需要将真实图像也归一化到[-1,1]才能匹配，否则判别器会学到区分0-1和-1-1.这是一个潜在问题。通常的做法是在数据预处理时用Normalize将图像从[0,1]转到[-1,1].本代码未做，可能影响训练效果。但许多DCGAN实现确实将真实图像归一化到[-1,1].
  • 判别器：隐藏层用LeakyReLU，避免ReLU的“死亡”问题（负值梯度为0），使判别器对生成图像也能提供有意义的梯度。负斜率0.2是经验值.

5. 损失函数与训练目标

  • 采用二分类交叉熵，判别器学习区分真实与生成，生成器学习欺骗判别器.

  • 训练过程可视为极小极大博弈：

  • 实际实现中，生成器常优化 $\log D(G(z))$而非 $\log (1-D(G(z)))$，以避免早期梯度饱和.

  • 本代码正是如此：$g_{loss} = criterions(output, true_{labels})$ 即最大化 $\log D(G(z))$.

6. 优化器与超参数

  • Adam优化器（lr=2e-4, $β_1$=0.5, $β_2$=0.999）是DCGAN作者推荐的组合。$β_1$取0.5而非默认0.9，可降低动量，使训练更稳定.
  • 批量大小216较大，有助于稳定判别器梯度，但需显存足够.

7. 训练技巧与潜在问题

  • 代码中每个iteration都训练一次判别器和一次生成器，比例1:1。实际训练时，若判别器太强，生成器梯度消失，可适当增加生成器训练次数（如判别器每训练k次，生成器训练1次）.
  • 没有使用标签平滑（如将真实标签设为0.9而非1），可能导致判别器过于自信，生成器梯度消失。添加标签平滑可提高鲁棒性.
  • 生成图像保存时使用了normalize=True，这会将Tensor值范围自动调整到[0,1]再保存，如果生成器输出是[-1,1]，此操作会将其归一化到[0,1]，保存的图像正常.
  • 真实图像预处理未归一化到[-1,1]可能影响训练，建议增加transforms.Normalize(mean=0.5, std=0.5)将[0,1]映射到[-1,1].

8. 推理脚本注意事项

  • 必须调用model.eval()关闭BN的训练行为，保证推理一致性.
  • 保存图像时只取第一张out_img.data[0]，也可保存多张.

总结

​ 这份代码实现了一个标准的DCGAN，结构清晰，注释详细。生成器和判别器采用全卷积架构，利用转置卷积和步长卷积实现上/下采样，批归一化和LeakyReLU等技巧提升了训练稳定性。通过交替优化两个网络，最终生成器能学会从随机噪声中产生逼真的人脸图像。训练过程中保存中间结果和模型权重，便于监控和继续训练。推理脚本可单独使用，加载已训练模型生成新图像.
